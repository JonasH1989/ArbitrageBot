# Arbitrage Bot - Feature Requests

## 1. SERVER & SECURITY (Phase 1)

### Move to Hetzner
- Performanter Server statt lokaler Maschine
- Vorausetzung für alles Weitere

### Authentication System
- SSL-Verschlüsselung hinter Domain
- User-Authentication mit Passwort
- 2FA (Google Authenticator ODER Email)
- Read-Only Accounts für Monitoring
  - User kann nur Daten sehen, nichts einstellen

### Mobile Access
- App-Zugriff via Smartphone
- iPhone und Android
- Authentifizierter Read-Only-Zugriff

## 2. MULTI-PAIR TRADING (Phase 2)

### Exchange Pairs Validierung
- Prüfe welche Coins auf BEIDEN Börsen gelistet sind
- Nur diesePairs können Arbitrage haben
- Aktuell: KuCoin + MEXC

### Mehr Exchanges einbinden
- WeBit (WeBit)
- Beinen (Binance)
- Weitere Börsen
- Alle mit API-Anbindung

---

## 3. COIN FILTERUNG (Phase 3)

### NICHT Major Coins
- Bitcoin, Ethereum, XRP sind zu groß
- Zu kleines Spread, zu großes Volumen
- Arbitrage lohnt sich nicht

### Fokus auf LOWCAPS!
- Extrem kleines Volume
- Extrem große Spreads
- Hier liegt die echte Opportunity!
- Masse an kleinen Trades = Coin-Akkumulation

---

## 4. AUTOMATISIERTE ANALYSE (Phase 4)

### Coin-Katalogisierung
- Scan alle Coins auf Exchange A
- Kreuze ab mit Exchange B
- Finde gemeinsame Pairs
- Automatisch katalogisieren

### Volatilität-Index
- Externe APIs für Volatilität
- Coins nach Volatilität bewerten
- Automatischer Algorithmus

### Opportunity-Scoring
- Ranking derPairs nach Profitabilität
- Automatische Filterung
